# LiLu — Взуттєва фабрика

> Веб-платформа для продажу взуття власного виробництва

**🌐 Живий сайт:** [https://3-course-project-demo.vercel.app](https://3-course-project-demo.vercel.app)
**🔧 Адмін-панель:** [https://3-course-project-demo.vercel.app/admin](https://3-course-project-demo.vercel.app/admin)

---

## Про проєкт

LiLu — повноцінний інтернет-магазин взуття, розроблений за три спринти:

| Sprint | Що зроблено |
|--------|-------------|
| Sprint 1 | Каталог, фільтри, кошик, оформлення замовлень, адмін-панель |
| Sprint 2 | Авторизація, особистий кабінет, live чат-підтримка |
| Sprint 3 | PostgreSQL, деплой на Vercel, мобільна адаптація |

---

## Функціонал

### Для покупця
- 🛍 Каталог з фільтрацією (категорія, колір, розмір, сезон, пошук)
- 🛒 Постійний кошик (зберігається між сесіями)
- 📦 Оформлення замовлення: Нова Пошта або кур'єр
- 👤 Особистий кабінет: замовлення, профіль, історія чату
- 💬 Чат-підтримка: FAQ-бот + живий оператор
- 🔗 Авторизація через телефон або email

### Для адміністратора
- 📋 CRUD товарів та управління залишками
- 📊 Управління замовленнями зі зміною статусів
- 👥 Управління користувачами та ролями
- 💬 Дашборд чату з відповідями в реальному часі

## Структура репозиторію

```
demo/
├── docs/            ← Технічна документація та звіти
├── Report/          ← Документи та звіти для інвесторів (Project Charter, Backlog, WBS, звіти спринтів тощо)
├── sprint1/         ← Каталог, кошик, адмін (v1)
└── sprint2/         ← Повна версія (production)
```

> Активна версія — `sprint2/`

## Проєктні звіти та артефакти (Report/)

У папці `Report/` містяться офіційні проєктні матеріали, що надавалися стейкхолдерам та інвесторам протягом усього процесу розробки:
- **LiLu_Project_Charter.docx** — Статут проєкту (опис цілей, бізнес-потреб та меж розробки).
- **LiLu_Product_Backlog.docx** — Беклог проєкту з деталізацією користувацьких історій (User Stories).
- **LiLu_WBS.docx** — Ієрархічна структура робіт (Work Breakdown Structure).
- **LiLu_Schedule_Baseline.docx** — Базовий розклад та календарний план проєкту.
- **LiLu_Risk.docx** — Аналіз ризиків та стратегія реагування на ризики.
- **LiLu_Quality_Management_Plan.docx** — План забезпечення та контролю якості (QA/QC).
- **LiLu_Comm_Plan.docx** — План комунікацій та взаємодії всередині команди та зі стейкхолдерами.
- **LiLu_Stakeholders_Matrix.md** — Матриця впливу та інтересів зацікавлених сторін.
- **LiLu_Sprint_1_Report.docx** — Звіт про виконання завдань Sprint 1.
- **LiLu_Sprint_2_Report.docx** — Звіт про виконання завдань Sprint 2.
- **LiLu_Sprint3_Report_Full.docx** — Детальний підсумковий звіт за результатами фінального Sprint 3.

## Локальний запуск

```bash
cd sprint2
npm install
npx prisma migrate dev
node prisma/seed.js
npm run dev
```

Потрібен файл `.env` з `DATABASE_URL`, `DIRECT_URL`, `AUTH_SECRET`, `NP_API_KEY`.
