# LiLu E-Commerce — Sprint 2: Огляд

**Дата:** квітень 2026 р.
**Гілка:** `sprint2`

---

## Що було зроблено

Sprint 2 — впровадження повноцінної системи авторизації, особистого кабінету користувача, адміністративних інструментів і live чат-підтримки.

---

## Нові можливості

### Для покупця

**Реєстрація і вхід**
- Реєстрація через телефон або email (або обидва одночасно)
- Підтвердження в реальному проєкті не потрібне (навчальна версія)
- Вхід за телефоном або email + пароль
- Сесія зберігається 7 днів (httpOnly cookie)

**Особистий кабінет** (`/account`)
- Вкладка **Замовлення** — повна історія з деталями, статусами, методами доставки
- Вкладка **Профіль** — редагування імені, телефону, email; зміна пароля
- Вкладка **Підтримка** — архів звернень до чат-підтримки з повними діалогами

**Розумна рекламація замовлень**
- Якщо покупець замовив товар як гість, а потім зареєструвався з тим самим телефоном або email — всі попередні замовлення автоматично прив'язуються до його акаунту

**Автозаповнення в кошику**
- Кнопка «Заповнити з профілю» на сторінці оформлення замовлення — підтягує ім'я, телефон та email

**Чат-підтримка**
- Floating кнопка 💬 на всіх сторінках
- 6 швидких питань (FAQ) з автовідповідями бота
- Підключення живого спеціаліста при потребі
- Гостьові чати очищаються при закритті вкладки; чати авторизованих — зберігаються

---

### Для адміністратора

**Управління користувачами** (`/admin/users`)
- Список всіх зареєстрованих користувачів
- Призначення ролі admin через інтерфейс

**Чат-дашборд** (`/admin/chat`)
- Всі відкриті та закриті звернення
- Повна переписка включно з FAQ-контекстом до підключення
- Відповіді в реальному часі (polling 3 с)
- Закриття / відновлення теми

## Технічні рішення

| Завдання | Рішення |
|----------|---------|
| Edge Runtime несумісний з Node.js `crypto` | Перехід на Web Crypto API (`crypto.subtle`) |
| `middleware.ts` застарів у Next.js 16 | Замінено на `proxy.ts` |
| Нова Пошта показувала лише 50 відділень | Ліміт збільшено до 500 |
| Дублювання повідомлень у чаті | `seenIds Set` — кожен ID повідомлення відстежується |
| Гостьові замовлення недоступні після реєстрації | Автоматична рекламація при реєстрації |
